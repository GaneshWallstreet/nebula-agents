# Lifecycle Gates Log — F####-{slug} run {run-id}

> Required per §8. Summary of validator/lifecycle invocations for this run. Sibling to `gate-decisions.md`; this file records what *ran*, gate-decisions records what *was decided*.

## Lifecycle Gate Run

Per-invocation rows below.

## Command

The command line invoked, including arguments.

## Stage

The lifecycle stage (`G0`/`G1`/.../`closeout`).

## Exit Code

Integer exit code from the invocation.

## Result

`PASS` / `FAIL` / `WARN`. May include `PASS WITH RECOMMENDATIONS` for governed signoff rows.

## Output References

Run-folder-relative paths to logs or report files generated by this invocation.

## Skipped Gates

If the run intentionally skipped a gate (e.g. non-runtime feature skipping G1), record the gate ID and the reason here.

## Example Block

```text
### G0
Command: python3 agents/product-manager/scripts/validate-feature-evidence.py --stage G0 --feature F0036 --run-id 2026-05-19-5ab6f922
Stage: G0
Exit Code: 0
Result: PASS
Output References: validation-summaries/g0.json
Skipped Gates: -
```
